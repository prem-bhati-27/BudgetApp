Identity Statement

The One Sentence

BudgetSplit tells you exactly where your money went and whether you're on pace — across your own spending, your shared tabs, your goals, and your bills.

Primary User
Urban Indian, 22–35. Daily UPI user. Splitting costs with flatmates, paying for group trips, managing their own grocery and subscription budget. Has never opened a spreadsheet for finances but absolutely checks their UPI app history.
Single Question on Open
"Am I on pace this month?"
This resolves to a single number (spent vs. budget pace) or a single verdict (on track / over pace). Every home screen element must directly support or contextualise this answer — not add noise alongside it.
Logo Animation
The entry logo animation is preserved entirely. No changes to onboarding animation or splash. The only onboarding addition: a single routing question after the animation — "Here to track your own spending, split with people, or both?" — that sets a soft preference, not a hard gate.

Home Screen Spec

The Hero: "₹X,XXX spent this month" — one SpaceMono XL number, always available regardless of setup, directly answering "how much have I spent?" Pace bar appears once a budget is set; health chip appears once 1+ periods of data exist. Both are graceful additions, never blockers.
Ranked Sections (descending importance)
01
Hero card — Spent this month
Data: getTransactionsInRange(me.id) → my share. Tap-through: category breakdown (same screen, scroll down). Tab pills: Today / Month / Year drive the number.
02
Balance strip — You owe / Owed
Data: getGlobalNet(). Tap "Settle →": opens unified SettleFlow sheet. Hidden when all zeros.
03
Where it went — top 3 categories
Data: same txn range aggregated by category. Bar width = % of total spend. Tap → category/[name] detail. "+ N more" collapses the rest.
04
Coming up — next 3 recurring items
Data: materializeDueOccurrences() + detected subs. Shows next payment name, amount, days until. Tap → Plan tab.
05
Health band — 1-line verdict
Data: computeHealthScore() — already computed from hero data. Renders amber/red/green chip + single worst-factor text. Tap → detail sheet (all 5 dimensions). Default-on, no flag gate.
❌ NOT on Home
Savings pool balance · Savings goal list · Full charts · Insights carousel · Income breakdown · Spending forecast · Audit log · Group-by-group budget bars · Financial health score as a full section · Tracking streak (demoted to Settings profile).

Primary Add Expense Flow

One tap. FAB → bottom sheet (SheetModal). No menu, no four options. The default mode is always "expense for me." Every other mode — income, itemized, split, recurring — is a progressive disclosure from inside the sheet.
1

Amount

Full-width numpad. SpaceMono 36px. No label — the ₹ symbol is the label. Amount field is auto-focused on sheet open.

2

Category + Date pills

Smart-matched from note text (learnedMatch → matchCategory). Tappable to override. Date defaults today. Both in one horizontal row below amount.

3

Note (optional)

Single text field. Drives smart category matching. Placeholder: "What was it for?"

★

Budget nudge row (NEW — Afford context)

Live: queries getCategoryBudget() and current spend in that category. Shows: "₹2,800 left in Food this month" in green / amber / red. No category budget → shows getCashPosition().available as safe-to-spend. Both null → row hidden. No navigation needed — afford check is answered inline.

4

Split selector

"Just me ▾" → taps to expand member picker + split type (equal / exact / percent / shares). Collapses back when no members selected.

5

▸ More options (progressive disclosure)

Single row below split: "Income · By items · Recurring". Each toggles a mode. Income: flips kind. By items: pushes itemized screen. Recurring: expands freq/end picker inline.

6

Log expense button

PrimaryButton full-width. On save: haptic success, sheet dismisses, home data refreshes. Duplicate detection: shows inline warning (not blocking alert).

Money Between People — Unified Flow

One model: "Settle up" is the single entry point for all money that moves between people. The distinction between "transfer" (you paid them directly) and "settle" (marking a split resolved) is an implementation detail — the user never chooses between them. The flow presents the decision they actually need to make: who, how much, and how it was paid.
Merge Plan
app/settle.tsx+SettleSheet.tsx+add/transfer.tsx
Merge into single SettleFlow bottom sheet. All three write the same kind:'settlement' txn via insertTxn().
Entry Points → All Open Same Sheet
Home → balance strip → "Settle →"
Groups → person balance row
Group detail → member balance
1

WHO + AMOUNT (pre-filled)

Pre-populated from context: simplify(getGlobalNet()) determines recommended person and amount. User can change person via picker or adjust amount (partial settle). Direction is stated plainly: "Rahul owes you ₹2,100" — no debit/credit ambiguity.

2

HOW (replaces transfer screen)

Three chips: 💵 Cash · 📱 UPI · 🏦 Bank. Optional note. This replaces add/transfer.tsx entirely — the payment method is the only decision left, and it's one tap.

3

CONFIRM + WRITE

Writes kind:'settlement' txn in the common group (uses existing getCommonGroupId()). If no shared group, surfaces the existing alert inline — not buried in async error. Haptic + dismiss. Balances update immediately.

Per-Screen State Matrix

Every kept screen ships all four states. No bare "No data yet" text. Use existing EmptyState, SkeletonCard, ErrorState components throughout.
Screen Empty Loading Error Full
Home EmptyState: "Add your first expense →" with FAB highlight pulse. Hero shows ₹0. Balance strip hidden. SkeletonCard for hero + 3 category rows. Tab pills still interactive. ErrorState with retry. Hero shows last cached value with "Offline" badge. All sections visible per hierarchy; sections with no data (e.g. no upcoming bills) are hidden — not empty placeholders.
Groups EmptyState: "Create a group to start splitting." + primary CTA button. 2× SkeletonCard at group card height. ErrorState with retry. No stale data shown. Groups listed, balances shown per group, budget bars where set.
Plan Three sections: goals empty → "Set a savings goal"; budgets empty → "Set a budget in a group"; upcoming empty → hidden section. SkeletonCard per section. ErrorState. Last cached pool balance shown if available. Pool card → goals → upcoming → what-if row. Insights shown if ≥2 periods of data.

Proposed Removals — Flagged for Owner Approval
None. Zero features removed. Every feature in the codebase has a home under the "where money goes" spine. The features that were described as "debt" in the teardown (health score, afford, forecast, subscription detection, savings insights, OCR) are relocated and wired, not cut. If after Phase 1 review the owner decides a specific feature is truly not useful, it should be flagged explicitly at that point — not as part of this spec.
